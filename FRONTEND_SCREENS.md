## 2단계: 부동산 카드 경매 화면 (Phase2Screen)

| # | GAME_RULES 요구사항 | 구현 여부 | 비고 |
|---|---------------------|-----------|------|
| 1 | 참가자들의 프로필, 이름 | ✅ | opponents 영역 |
| 2 | 참가자들이 제출한 카드 여부 (제출 전/후) | ✅ | "제출 완료" / "고민 중..." |
| 3 | 카드 제출에 남은 시간 | ✅ | `useTimeLeft`, phase2StartTime |
| 4 | 라운드 숫자 텍스트 | ✅ | `gameState.phase2RoundNumber` |
| 5 | 참가자 수에 맞게 펼쳐진 부동산 카드들 | ✅ | `gameState.currentRealEstateCards` |
| 6 | 아직 펼쳐지지 않은 부동산 카드 뭉치 | ✅ | "부동산 카드" 덱 |
| 7 | 내 프로필 | ✅ | 하단 내 정보 |
| 8 | 내 남은 돈 | ✅ | `me?.coins` |
| 9 | 내가 가진 직업 카드들 (선택 가능) | ✅ | JobCardMini 클릭 제출 |
| 10 | 내가 획득한 부동산 카드 목록 | ✅ | "내 부동산" 영역 |
| — | 아이템 사용 (리롤/엿보기) | ✅ | 2단계는 리버스 미지원 |
| — | 제출 후 카드 동시 오픈 연출 | ✅ | `allPlayersSelected` 시 카드 공개 |

**규칙 준수**

- 제한 시간 내 미선택 시 **가장 낮은 직업 카드 자동 제출** — 서버 처리 필요, 프론트는 타이머 표시만.
- 부동산 카드 배분(높은 직업 → 높은 부동산) 및 동점 시 턴 순서 — 서버 로직, 프론트는 결과 반영만.

| # | GAME_RULES 요구사항 | 구현 여부 | 비고 |

## 화면 전환 (App.tsx)

- **로비** → 방장 게임 시작 → **아이템 선택** → **1단계** → **2단계** → **결과**
- 서버 `phase` / `gameState`에 따라 `item_selection`, `playing`, `game_over`로 화면 전환.
- 결과 화면에서 "로비로 돌아가기" 시 홈(`home`)으로 이동.

## 공통 확인 사항

1. **타이머**: 서버 기준 시간(`turnStartTime`, `phase2StartTime`)으로 남은 시간 계산. 만료 시 서버가 자동 포기/자동 카드 선택 처리.
2. **비공개 정보**: 다른 참가자의 남은 돈·직업 카드는 노출하지 않음. 엿보기 사용 시에만 일회성으로 표시.
3. **아이템 공개**: 누가 어떤 아이템을 선택했는지는 모든 참가자에게 보이도록 구현됨 (Phase1/Phase2 참가자 카드에 아이콘).
